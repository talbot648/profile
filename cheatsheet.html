Variables


A variable is a piece of data with an assigned value.
Variables can be declared using a keyword called "var" before declaring the variable a name.
The variable must be assigned to a type which is either a string, integer or boolean
To assign a variable to a value, you have to state the variable and then equals
"Input the value". This has to match the assigned type that was previously stated.

Short assignment operator


To make our lives easier, we can use the short assignment operator := which is used to declare and initialise a variable in a single line.
It is useful when you don't want to fill up your code with "Var .... int/string"

If statements


An if statement is used to check whether a certain condition is true. If so it will run a certain piece of code.
However, if it is false, we can use an else statement to run a different piece of code.

For loops


A for loop is used to do something repeatedly without having to write the code over and over again.
For example, you have a box full of books and you want to take a book out one at a time until all
of the books are taken out. You would use a for loop.

Functions


A function is an independent section of code that is used for performing a specific task given from the user. You can picture them as a container.
Functions take in parameters which are variables that live inside of the function and can also output these parameters to different functions.

"go mod init"

The "go mod init" command in short creates a go.mod file for you and initialises a new module. This file also downloads the modules
you use in your projects. If you don't have this, you cannot add packages to your progams.

Running Go code


There are a couple ways to run Go code.
Firstly, you can open up a new terminal and select command line. Then you simply type in "go run ." You could also open up command prompt, navigate to your go code file and run it from there using "go run ."

Why we automate testing of code Example of GO test and how to execute it
Firstly, automating the testing of code allows you to stay in a flow state and save time by
not needing to manually test each section of code.
Another benefit of automating testing is if you have a large piece of code with a wide variety of
potential outcomes it saves so much time.

Another reason why we automate testing is that the test will actually tell you what line went wrong
and what was expected. This speeds up time taken to fix an error. Also if you are working with a team
it helps everyone see that the code they're interacting with works as executed.		 package main
import "testing"

func TestHello(t *testing.T) {
t.Run ("saying hello to people", func(t *testing.T) {
got := Hello("Chris","")
want := "Hello, Chris"
assertCorrectMessage(t, got, want)
})
}
